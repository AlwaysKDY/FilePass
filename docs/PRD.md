# FilePass — 极速局域网跨端传输工具 · 产品需求文档 (PRD V1.1)

> 最后更新：2026-04-21

---

## 一、项目概述

### 1.1 项目背景

| 维度 | 现状痛点 | FilePass 方案 |
|------|---------|--------------|
| 传输链路 | 需打开手机助手 / 微信，经外网中转 | 局域网直连，零中转 |
| 空间占用 | 微信缓存动辄数 GB | 双端合计 < 60 MB 内存 |
| 操作步骤 | 至少 4 步：打开App → 选文件 → 发送 → PC端打开 | 1 步：下拉通知栏点击 Quick Tile |
| 游戏干扰 | 传文件时弹窗抢焦点 | 绝不弹窗，托盘气泡静默提示 |

### 1.2 核心目标

1. **极轻量**：PC 端常驻内存 < 50 MB，手机端 APK < 5 MB。
2. **极速入口**：手机端通过 Android Quick Settings Tile 一键推送剪贴板；通过系统 Share Sheet 一键发送文件。
3. **高性能**：后台 CPU 占用 ≈ 0%；局域网直连，传输速度仅受路由器上限约束。
4. **零干扰**：PC 端绝不抢焦点，不影响游戏/全屏应用。

### 1.3 用户画像

- **唯一用户**：自用，Vivo 手机（OriginOS）+ Windows 10 台式机。
- **主要场景**：
  - 复制手机浏览器内容 → 粘贴到 PC 编辑器。
  - 手机截图 / 照片 → 快速传到 PC 桌面。
  - PC 剪贴板文本 → 推送到手机（V2 规划）。

## 二、用户场景 & 使用流程

### 场景 A：推送剪贴板文本

```
手机复制文本 → 下拉通知栏 → 点击「FilePass」Quick Tile
→ 手机读取剪贴板 → HTTP POST /api/text → PC 写入剪贴板
→ PC 托盘气泡「已接收文本 (128字)」
→ 用户在 PC 直接 Ctrl+V
```

### 场景 B：推送文件 / 图片

```
手机相册/文件管理器 → 长按文件 → 分享 → 选「FilePass」
→ HTTP POST /api/file (multipart) → PC 保存至 ~/Desktop/FilePass_Received/
→ PC 托盘气泡「已接收 photo_001.jpg (2.3MB)」
```

### 场景 C：PC → 手机（V2 规划）

```
PC 托盘右键 → 「推送剪贴板到手机」
→ HTTP POST /api/phone/text → 手机通知栏显示内容
```

## 三、功能需求清单

### 3.1 PC 端（Server 角色）

| 编号 | 模块 | 功能描述 | 优先级 |
|------|------|---------|--------|
| S-01 | 静默启动 | 启动后自动获取局域网 IP，绑定 `0.0.0.0:8765`，无主窗口 | P0 |
| S-02 | mDNS 广播 | 使用 zeroconf 注册 `_filepass._tcp.local.`，手机自动发现 | P0 |
| S-03 | 系统托盘 | 右下角托盘图标（pystray），右键菜单：IP信息 / 接收目录 / 退出 | P0 |
| S-04 | 文本接收 API | `POST /api/text` → 写入 Windows 剪贴板（pyperclip） | P0 |
| S-05 | 文件接收 API | `POST /api/file` (multipart) → 保存至 `~/Desktop/FilePass_Received/` | P0 |
| S-06 | 托盘通知 | 接收成功 → 托盘气泡提示（Windows 10 Toast），绝不弹窗抢焦点 | P0 |
| S-07 | 开机自启 | 注册 Windows 启动项（注册表或快捷方式到 Startup 文件夹） | P1 |
| S-08 | 简易认证 | 首次配对生成 Token，写入双端配置，后续请求携带 Token 校验 | P1 |
| S-09 | 反向推送 | `GET /api/clipboard` 供手机拉取 PC 剪贴板内容 | P2 |

### 3.2 手机端（Client 角色）

| 编号 | 模块 | 功能描述 | 优先级 |
|------|------|---------|--------|
| C-01 | Quick Settings Tile | 下拉通知栏添加自定义 Tile，点击即推送当前剪贴板到 PC | P0 |
| C-02 | mDNS 自动发现 | 使用 Android NsdManager 自动搜索 `_filepass._tcp.local.` | P0 |
| C-03 | Share Sheet 注册 | 注册为系统分享目标，支持文本、图片、任意文件分享至 PC | P0 |
| C-04 | 极简主界面 | 仅显示连接状态、PC 地址、手动输入 IP 备用入口 | P0 |
| C-05 | 后台保活 | 轻量 Foreground Service + WorkManager 保持 Quick Tile 可用 | P1 |
| C-06 | 拉取 PC 剪贴板 | 从 `/api/clipboard` 获取 PC 内容，写入手机剪贴板 | P2 |

## 四、非功能需求

### 4.1 性能指标

| 指标 | 要求 |
|------|------|
| PC 端内存占用 | 常驻 ≤ 50 MB（pyinstaller 打包后） |
| PC 端 CPU 占用（空闲） | ≈ 0.00%（异步 IO，无轮询） |
| 手机端 APK 体积 | ≤ 5 MB |
| 文本传输延迟 | < 200 ms（局域网） |
| 文件传输吞吐 | ≥ 50 MB/s（千兆局域网理论值） |
| mDNS 广播开销 | ~100 Bytes UDP / 数秒，对游戏延迟无影响 |

### 4.2 可靠性

- mDNS 发现失败时，提供手动输入 IP 的降级方案。
- 文件传输支持大文件（分块上传 + 进度回调），避免内存爆炸。
- 传输失败时，手机端 Toast 提示具体原因。

### 4.3 安全性

- **Token 认证**：首次配对后，所有 HTTP 请求必须携带 `Authorization: Bearer <token>`。
- **仅局域网**：Server 绑定 `0.0.0.0` 但建议路由器防火墙限制外网访问 8765 端口。
- **文件名清洗**：接收文件名进行安全过滤，防止路径穿越攻击。
- **文件大小限制**：单文件默认上限 500 MB，可在配置中调整。

### 4.4 兼容性

- PC：Windows 10 21H2+，Python 3.10+。
- 手机：Android 10+（API 29+），兼容 Vivo OriginOS 的后台策略。

## 五、约束与依赖

### 5.1 网络环境

- 双端必须在 **同一局域网** 内。
- 若使用 Clash 等代理，需将 `192.168.*` 和 `*.local` 加入绕行规则。

### 5.2 Android 限制

- **Android 10+ 剪贴板限制**：后台应用无法读取剪贴板，需通过 Quick Tile（前台交互触发）或辅助功能绕过。此为核心技术挑战。
- **Vivo OriginOS 后台限制**：需引导用户关闭 FilePass 的电池优化，添加自启动白名单。

### 5.3 不做清单

- ❌ 不做外网穿透 / 云中转。
- ❌ 不做聊天 / 消息记录功能。
- ❌ 不做多设备支持（仅 1 手机 + 1 PC）。
- ❌ PC 端不做 GUI 主窗口。

## 七、验收标准

1. 手机复制文本 → 下拉点击 Tile → PC 剪贴板已更新，全程 < 1 秒。
2. 手机分享一张 5 MB 图片 → PC 桌面出现文件，全程 < 3 秒。
3. PC 全屏游戏中接收文件 → 无弹窗、不闪退、不抢焦点。
4. 双端重启后自动重连，无需手动配置。
